¿Como se desarrolla software?

Por ejemplo: Gestionar sus clientes.

Entonces...

  • Para que un software sea:
    • Util

Hay al menos 3 tipos de personas

  • Roles:
    • Usuario: Quien tiene la necesidad
    • Desarrollador: Quien transforma esa necesidad en un programa
    • Administrador del sistema: Quien se encarga que el sistema este funcionando correctamente todo el tiempo

Hay al menos 5 componentes

¿Por que es complicado hacer software?

Porque hay más de 3 personas

  • Hay muchas personas implicadas en el proceso
    • Varias Usuarios del sistemas
    • Varios Desarrolladores
    • Varios Administradores

Porque hay más de 3 Roles

  • Muchos roles por actividades y tecnologias
    • Analista Funcional
    • QA
    • PM
    • Frontend
    • Backend
    • Web Dev Fullstack

Porque se introducen constantemente nuevos cambios

El dificil lograr la simplicidad de diseño

Todo cambio puede provocar nuevos errores

  • Por errores de comunicación
    • El usuario interpreta/explica una cosa
    • El desarrollador interpreta/entiende otra
    • Muchos más...
  • Exceso de confianza y prejuicios
    • Pensar que es simple
    • Pensar que se puede hacer rapido
    • Reinventar la rueda
    • etc, etc

¿Que es DevOps?
  • Es una cultura de trabajo
  • Con técnicas y buenas prácticas
  • Permite agilizar los proceso de entrega de software
  • Garantiza la calidad del software entregado
  • Es tolerante a fallos
La problematica

¿Por que surge todo esto?

El area de desarrollo

Introduce constantemente cambios

  • Agrega nuevas caractericas a un sistema
  • Corrigue errores en versiones anteriores

El area de administración de sistemas

Constantemente estabiliza los sistemas

  • Necesita estabilidad en producción.
    • Los usuario utilizaran los sistema para su jornada diaria
  • Si el sistema funciona. No hay que cambiarlo!!
¿Como abordar la problematica?

NO HAY QUE DUPLICAR
